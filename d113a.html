2. 수업을 여는 멘트

여러분은 온라인 쇼핑몰, 메신저, 학교 학습 플랫폼, 게임 계정에 로그인할 때 아이디와 비밀번호를 사용합니다. 또 스마트폰에는 사진, 위치 기록, 연락처, 결제 정보가 저장되어 있습니다. 이런 데이터가 편리함을 주는 동시에 잘못 노출되면 큰 피해를 만들 수 있습니다. 오늘은 데이터를 분석하고 활용하기 전에 반드시 알아야 하는 데이터의 보안을 배웁니다.
Core Concepts

3. 각각의 개념에 대한 상세 설명

개념 1. 데이터 보안

데이터 보안은 데이터를 허가되지 않은 접근, 변경, 삭제, 유출, 오용으로부터 보호하는 활동입니다. 데이터 과학과 머신러닝에서는 많은 데이터를 다루므로 보안은 선택이 아니라 필수입니다.

개념 2. 개인정보와 민감정보

개인정보는 이름, 전화번호, 주소, 학교, 이메일처럼 특정 개인을 알아볼 수 있는 정보입니다. 민감정보는 건강, 생체정보, 유전정보처럼 노출될 경우 큰 피해나 차별을 만들 수 있는 정보입니다.

개념 3. 보안 3요소

기밀성은 허가된 사람만 데이터에 접근하는 것, 무결성은 데이터가 허가 없이 바뀌지 않는 것, 가용성은 필요한 사람이 필요할 때 데이터를 사용할 수 있는 것을 뜻합니다.

\\[ \\text{데이터 보안} = \\text{기밀성} + \\text{무결성} + \\text{가용성} \\]

개념 4. 인증

인증은 사용자가 누구인지 확인하는 과정입니다. 비밀번호, OTP, 지문, 얼굴 인식, 2단계 인증이 대표적인 인증 방법입니다.

개념 5. 접근 권한

접근 권한은 사용자가 어떤 데이터를 볼 수 있고 수정할 수 있는지를 정하는 규칙입니다. 학생은 자신의 성적만 보고, 교사는 담당 학급의 성적을 볼 수 있습니다.

개념 6. 암호화

암호화는 데이터를 알아볼 수 없는 형태로 바꾸어 허가되지 않은 사람이 내용을 읽지 못하게 하는 방법입니다.

개념 7. 해시

해시는 데이터를 고정 길이의 값으로 바꾸는 방법입니다. 비밀번호 저장에 자주 사용되며, 원래 값을 되돌리기 어렵다는 특징이 있습니다.

개념 8. 익명화와 마스킹

익명화는 개인을 알아볼 수 없도록 정보를 제거하거나 바꾸는 방법입니다. 마스킹은 일부 정보를 별표 등으로 가리는 방법입니다.

개념 9. 백업

백업은 데이터가 손상되거나 삭제될 경우를 대비해 복사본을 안전한 곳에 저장하는 것입니다.

개념 10. 데이터 보안 윤리

데이터 과학자는 동의, 목적 제한, 최소 수집, 안전한 보관, 공정한 활용을 고려해야 합니다.

Python Codes

4. 각각의 개념과 상세 설명 관련 파이썬 코드

코드 1. 개인정보 마스킹

phone = "010-1234-5678"
masked = phone[:4] + "****" + phone[8:]
print(masked)

설명: 전화번호의 가운데 자리를 별표로 가려 개인정보 노출을 줄입니다.

코드 2. 이메일 아이디 일부 가리기

email = "student123@example.com"
name, domain = email.split("@")
masked_email = name[:3] + "***@" + domain
print(masked_email)

설명: 이메일 전체를 그대로 보여 주지 않고 일부만 표시하면 개인정보 보호에 도움이 됩니다.

코드 3. 비밀번호 해시 만들기

import hashlib
password = "my_password_123"
hashed = hashlib.sha256(password.encode()).hexdigest()
print(hashed)

설명: 해시는 비밀번호를 원문 그대로 저장하지 않기 위한 기본 보안 방법입니다.

코드 4. 비밀번호 확인

import hashlib
saved_hash = hashlib.sha256("abc123".encode()).hexdigest()
input_password = "abc123"
input_hash = hashlib.sha256(input_password.encode()).hexdigest()

if input_hash == saved_hash:
    print("로그인 성공")
else:
    print("로그인 실패")

설명: 실제 비밀번호를 비교하지 않고 해시값을 비교하여 인증 여부를 확인합니다.

코드 5. 접근 권한 검사

role = "student"

if role == "teacher":
    print("전체 성적 데이터 접근 가능")
elif role == "student":
    print("본인 성적만 접근 가능")
else:
    print("접근 불가")

설명: 사용자 역할에 따라 접근 가능한 데이터 범위를 다르게 설정할 수 있습니다.

코드 6. 간단한 시저 암호

text = "DATA"
shift = 3
result = ""

for ch in text:
    result += chr((ord(ch) - ord("A") + shift) % 26 + ord("A"))

print(result)

설명: 문자를 일정한 칸만큼 이동해 암호문을 만드는 간단한 암호화 예제입니다.

코드 7. 보안 위험 단어 탐지

message = "내 비밀번호는 1234입니다"
danger_words = ["비밀번호", "주민등록번호", "계좌번호"]

for word in danger_words:
    if word in message:
        print("주의: 민감한 정보가 포함될 수 있습니다.")

설명: 메시지에 민감한 단어가 포함되어 있는지 확인하여 보안 위험을 줄일 수 있습니다.

Real Life Coding

7. 실생활 관련 코드 예제 5개 + 상세설명

예제 1. 전화번호 마스킹

phones = ["010-1234-5678", "010-9876-5432"]
for phone in phones:
    print(phone[:4] + "****" + phone[8:])

개인정보 보호전화번호를 화면에 표시해야 할 때 전체를 보여 주지 않고 일부를 가리면 개인정보 노출 위험을 줄일 수 있습니다.

예제 2. 이메일 마스킹

emails = ["minsu@example.com", "jiwoo@school.kr"]
for email in emails:
    name, domain = email.split("@")
    print(name[:2] + "***@" + domain)

계정 보안이메일 주소는 로그인 정보와 연결될 수 있으므로 일부만 표시하는 것이 안전합니다.

예제 3. 비밀번호 강도 검사

password = "abc12345"

if len(password) >= 8 and any(ch.isdigit() for ch in password):
    print("사용 가능한 비밀번호")
else:
    print("더 강한 비밀번호가 필요합니다.")

인증 보안비밀번호 길이와 숫자 포함 여부를 검사하여 너무 약한 비밀번호 사용을 줄일 수 있습니다.

예제 4. 역할 기반 접근 권한

users = {
    "teacher": "전체 성적 열람 가능",
    "student": "본인 성적만 열람 가능",
    "guest": "열람 불가"
}
role = "student"
print(users.get(role, "알 수 없는 사용자"))

접근 제어사용자 역할에 따라 데이터 접근 범위를 다르게 설정하면 불필요한 정보 노출을 막을 수 있습니다.

예제 5. 메시지 속 민감정보 경고

message = "내 계좌번호와 비밀번호를 보낼게"
danger_words = ["비밀번호", "계좌번호", "주민등록번호"]

warning = False
for word in danger_words:
    if word in message:
        warning = True

if warning:
    print("주의: 민감정보가 포함되어 있을 수 있습니다.")
else:
    print("민감정보 단어가 발견되지 않았습니다.")

메신저 보안메시지에 민감한 단어가 포함되어 있는지 확인하여 개인정보를 실수로 전송하는 상황을 예방할 수 있습니다.

8. 수업을 마무리하는 멘트

오늘 우리는 데이터 보안이 단순히 비밀번호를 어렵게 만드는 일이 아니라, 데이터를 안전하고 책임 있게 활용하기 위한 기본 조건임을 배웠습니다. 컴퓨터과학은 인증, 접근 권한, 암호화, 해시 같은 기술을 제공하고, 수학은 암호와 해시의 논리적 기반을 제공합니다. 과학은 신뢰할 수 있는 데이터를 보호하고 검증하는 태도를 강조합니다. 데이터 과학과 머신러닝은 많은 데이터를 다루기 때문에 보안과 윤리는 반드시 함께 배워야 합니다.

컴퓨터과학과의 연결

인증, 접근 권한, 암호화, 해시, 백업 시스템은 컴퓨터 보안의 핵심 기술입니다.

수학과의 연결

암호화와 해시는 규칙, 함수, 변환, 경우의 수, 논리적 추론과 깊이 연결됩니다.

과학과의 연결

데이터의 신뢰성과 무결성을 지켜야 실험과 관찰 결과를 올바르게 검증할 수 있습니다.

머신러닝과의 연결

머신러닝 데이터에는 개인정보가 포함될 수 있으므로 익명화, 권한 관리, 안전한 저장이 중요합니다.
